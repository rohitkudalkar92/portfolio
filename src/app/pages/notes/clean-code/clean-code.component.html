• Meaningful Names: Use descriptive variable and function names
  • Small Functions: Keep functions under 20 lines
  • Avoid Deep Nesting: Maximum 3 levels of indentation
  • Single Responsibility: Each function should do one thing
  • DRY Principle: Don't Repeat Yourself - eliminate code duplication
  • Error Handling: Handle errors gracefully
  • No Magic Numbers: Use named constants
  • Consistent Formatting: Follow style guidelines
  • Variables: Use nouns (user_name, total_price)
  • Functions: Use verbs (calculate_tax, send_email)
  • Booleans: Use is/has/can (is_valid, has_permission)
  • Constants: Use UPPER_CASE (MAX_RETRY_COUNT)
  • Copy-paste code: Same logic appears in multiple places
  • Similar functions: Functions that do almost the same thing
  • Repeated validation: Same validation logic in multiple methods
  • Duplicate constants: Same values defined in multiple files
  • Python: Follow PEP 8 (4 spaces, snake_case)
  • JavaScript: Use ESLint/Prettier (2 spaces, camelCase)
  • Java: Follow Oracle conventions (camelCase, braces)
  • General: Be consistent within your project
