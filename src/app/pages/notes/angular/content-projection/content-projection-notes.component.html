Content projection (also known as transclusion) allows you to insert content from a parent component into a child component's template using ng-content. It enables flexible and reusable component composition.

Use ng-content to project content into a component:
  • ng-content: Acts as a placeholder for projected content
  • Single Slot: All content between component tags is projected
  • No Selector: Without select attribute, accepts any content
  • Composition: Enables flexible component reusability

Combine ng-content with Angular control flow:
  • Projected content is always instantiated, even if not displayed
  • Use @if to control visibility, not instantiation
  • Content lifecycle is managed by parent component
  • Projected components maintain their state when hidden/shown

Use templates for more control over projected content:
  • @ContentChild: Query single projected element or directive
  • @ContentChildren: Query multiple projected elements
  • ngAfterContentInit: Lifecycle hook when content is initialized
  • ngAfterContentChecked: Hook after content change detection
  • Layout Components: Cards, panels, modals with flexible content
  • Wrapper Components: Adding common functionality around content
  • List Components: Custom rendering of list items
  • Tab Components: Multiple content areas with navigation
  • Dialog Components: Header, body, footer composition
  • Form Components: Custom form controls with labels
  • {{ practice }}
  • Projected content is always created, even if ng-content is in @if block
  • Cannot bind to projected content directly from child component
  • Styling projected content requires ::ng-deep or global styles
  • Content queries are available only after ngAfterContentInit
